# תיקון שליחת מיילים ב-AI Experts

## הבעיה
המערכת מנסה לשלוח מיילים דרך Resend אבל מקבלת שגיאת 403 (Forbidden) כי:
1. כתובת השולח היא `onboarding@resend.dev` - כתובת sandbox של Resend
2. במצב sandbox, ניתן לשלוח מיילים רק לכתובות מאומתות

## פתרונות אפשריים

### פתרון 1: אימות כתובות מייל (מהיר - למצב testing)
1. היכנס ל-Resend Dashboard: https://resend.com/emails
2. לך ל-Settings > API Keys
3. הוסף את כתובות המייל של המומחים שלך כ-"Verified emails"
4. כעת המערכת תוכל לשלוח מיילים לכתובות אלו

**חסרונות**: צריך לאמת כל כתובת מייל ידנית, לא מתאים לפרודקשן

### פתרון 2: אימות דומיין (מומלץ לפרודקשן)
1. היכנס ל-Resend Dashboard
2. לך ל-Domains
3. לחץ על "Add Domain"
4. הוסף את הדומיין שלך: `ai-experts.co.il`
5. הוסף את רשומות ה-DNS שResend מציג (TXT, DKIM, וכו')
6. המתן לאימות (עד 48 שעות, בדרך כלל מהיר יותר)
7. עדכן את `src/lib/mail.ts` בשורה 8:
   ```typescript
   const FROM_EMAIL = 'AI Experts <noreply@ai-experts.co.il>';
   ```

**יתרונות**: 
- שליחת מיילים ללא הגבלה
- נראה מקצועי יותר
- אמינות גבוהה יותר (SPF, DKIM)

### פתרון 3: השבתת מיילים זמנית (אם לא דחוף)
אם אתה לא צריך מיילים כרגע, אפשר פשוט להשאיר את זה כמו שזה - המערכת תמשיך לעבוד, רק המיילים לא יישלחו.

## מה קורה עכשיו?
- ✅ יצירת tasks עובדת מצוין
- ✅ הנתונים נשמרים ב-Supabase
- ❌ מיילים למומחים לא נשלחים (שגיאה 403)
- ✅ המערכת לא קורסת בגלל זה

## המלצה שלי
אם אתה רוצה שהמערכת תהיה מוכנה לפרודקשן - לך על **פתרון 2** (אימות דומיין).
זה ייקח 10-15 דקות להגדיר ואז הכל יעבוד חלק.

## איך לבדוק שהמיילים עובדים?
אחרי שתתקן (פתרון 1 או 2), צור task חדש ובדוק את הלוגים:
```bash
# בדוק את הלוגים ב-Cloud Console
https://console.cloud.google.com/logs/query?project=ai-experts-co-il
# חפש: "Email sent successfully"
```
